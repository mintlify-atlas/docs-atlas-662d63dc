---
title: "Get started with Multiagente in under five minutes"
description: "Go from zero to a working bot: install Bun and the ADK CLI, start the dev server, test all five specialist domains, and deploy to Botpress Cloud."
---

This guide walks you through everything you need to run Multiagente locally and deploy it to Botpress Cloud. By the end you will have a working bot that routes messages to the HR, IT, Sales, Finance, and Docs specialist agents and escalates to human support when needed.

<Steps>
  <Step title="Install Bun and the ADK CLI">
    Multiagente uses [Bun](https://bun.sh) as its runtime and package manager, and the Botpress ADK CLI (`adk`) to run and deploy the agent.

    Install Bun by following the instructions at [bun.sh](https://bun.sh). For the ADK CLI, follow the installation instructions in the [Botpress ADK documentation](https://botpress.com/docs/adk).

    Confirm both tools are available:

    ```bash
    bun --version
    adk --version
    ```
  </Step>

  <Step title="Clone the repo and install dependencies">
    Clone the Multiagente repository and install its dependencies with Bun:

    ```bash
    git clone https://github.com/alcorher/multiagente.git
    cd multiagente
    bun install
    ```

    The project depends on `@botpress/runtime` and `llmz`. Bun resolves and installs both from the lockfile in seconds.
  </Step>

  <Step title="Start the dev server">
    Run the dev server with hot reload. Any change you make to source files is picked up automatically without restarting.

    ```bash
    bun run dev
    ```

    This runs `adk dev` under the hood. The agent starts on the default model `cerebras:gpt-oss-120b` with two channels enabled:

    | Integration | Version |
    |---|---|
    | `chat` | `0.7.3` |
    | `webchat` | `0.3.0` |

    Leave the dev server running in this terminal and open a second terminal for the next step.
  </Step>

  <Step title="Test with adk chat">
    With the dev server running, open a second terminal and start a chat session:

    ```bash
    adk chat
    ```

    <Tip>
      `adk chat` connects directly to the running dev server and lets you send messages without a browser or external client. Use it to verify routing and HITL behavior before deploying.
    </Tip>

    Send a test message to each specialist domain to confirm routing works end-to-end:

    <CodeGroup>

    ```bash HR
    # Vacation and leave
    What's my vacation balance?
    ```

    ```bash IT
    # Password and access
    Reset my password
    ```

    ```bash Sales
    # Products and quotes
    What are the current promotions?
    ```

    ```bash Finance
    # Expenses and budgets
    Submit an expense report
    ```

    ```bash Docs
    # Botpress platform questions
    How do I add an integration?
    ```

    </CodeGroup>

    The orchestrator routes each message to the matching specialist and returns its result as a single reply.

    <Note>
      To test the HITL flow, ask to speak to a human ("I need to talk to a real person"). The bot activates HITL mode and pauses AI responses. To hand control back to the bot, send one of the resume keywords: `resume bot` or `volver al bot`.
    </Note>
  </Step>

  <Step title="Deploy to Botpress Cloud">
    When you are ready to go live, deploy the agent with a single command:

    ```bash
    bun run deploy
    ```

    This runs `adk deploy`, which builds the project and pushes it to Botpress Cloud. You can also run the build step separately before deploying:

    ```bash
    bun run build
    bun run deploy
    ```

    | Script | Command |
    |---|---|
    | `bun run dev` | `adk dev` |
    | `bun run build` | `adk build` |
    | `bun run deploy` | `adk deploy` |
  </Step>
</Steps>
